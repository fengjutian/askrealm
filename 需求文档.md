# 🎬 中国影视角色风格问答 App — 需求文档

## 1. 项目概述

一个 Flutter 移动端应用，用户以文字提问，App 以**中国影视/文学经典角色**的口吻、性格和语言风格进行回答，带来沉浸式互动体验。

**核心亮点：支持「单人对话」和「多人聊天室」两种模式** — 一个问题，多个角色同时回答，宛如角色们在同一个聊天室里各抒己见。

---

## 2. 目标用户

- 喜欢中国影视/文学、希望与经典角色"对话"的用户
- 想看不同角色针对同一个话题"同台互怼/讨论"的趣味用户
- 对 AI 对话产品有兴趣的体验用户

---

## 3. 核心功能

### 3.1 两种对话模式

#### 模式一：单人对话（1v1）
- 用户选择一个角色，进行一对一问答
- 角色保持人设连续对话

#### 模式二：多人聊天室（1vN）
- 用户选择 **2~4 个角色** 加入同一个聊天室
- 用户发送一个问题，**每个角色依次以自己的风格回答**
- 角色之间可以看到彼此的回答（通过上下文带入），营造"群聊"氛围
- 支持**追问**：用户继续发言，所有角色再次回复

## 4. 回答生成方案

### 方案：调用大模型 API ✅（已确认）
- 通过兼容 OpenAI API 格式的接口（如 DeepSeek、OpenAI 等）
- 接口地址和 API Key 由用户在设置页自行配置
- **单人模式**：携带**角色人设 System Prompt** + 对话历史 → 一次 API 调用
- **聊天室模式**：携带**全体角色人设** + 对话历史 → **N 次 API 调用**（每个角色独立调用，保证风格纯正）
- 支持自定义 API Base URL（兼容不同服务商）

### API 请求格式
```
POST {base_url}/v1/chat/completions
Headers:
  Authorization: Bearer {api_key}
  Content-Type: application/json

Body:
{
  "model": "gpt-4o-mini" (或用户指定),
  "messages": [
    {"role": "system", "content": "你是孙悟空..."},
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ],
  "temperature": 0.8
}
```

## 6. UI/UX 要求

### 6.1 页面结构

```
应用入口
  ├── 🏠 首页（模式选择）
  │   ├── 「单人对话」→ 角色选择页（单选）→ 对话页（1v1）
  │   └── 「多人聊天室」→ 角色选择页（多选 2~4）→ 聊天室页（1vN）
  └── ⚙️ 设置页（API Key / Base URL / Model 配置）
```

### 6.2 单人对话页
- 顶部 Bar：当前角色名 + Emoji 头像 + 切换角色按钮
- 消息列表：角色消息居左（带头像）、用户消息居右
- 底部输入区：文本框 + 发送按钮
- 空状态引导："选择一个角色，开始对话"

### 6.3 聊天室页（多人模式）
- 顶部 Bar：显示所有成员的小头像排列 + 房间名（如"三国演疫"）
- 消息列表：
  - 用户消息：靠右
  - 每个角色的回复：靠左，**带头像、角色名、彩色标签**，按顺序依次排列
  - 一条用户问题 → N 条角色回复（逐条出现，有加载动画）
- 底部输入区：文本框 + 发送按钮
- 输入指示器："孙悟空 正在输入…" / "诸葛亮 正在输入…"

### 6.4 角色选择页
- 卡片网格布局（2列），每张卡片包含角色 Emoji、名称、作品
- **单人模式**：点击即进入对话
- **聊天室模式**：可多选（2~4 个），选完后点"开始聊天"
- 已选角色有勾选标记

### 6.5 设计风格
- 暗色主题为主（影感沉浸）
- 角色用 Emoji 头像（初期不接入真实图片）
- 圆角卡片、柔和阴影
- 聊天室消息用不同**左侧色条 / 角色名标签色**区分
- 字体：中文字体优雅清晰

## 8. 数据模型

```dart
class Character {
  final String id;
  final String name;
  final String from;          // 出自作品
  final String emoji;         // 头像 Emoji
  final String description;   // 性格描述
  final String systemPrompt;  // 角色人设指令（给 AI 的 System Prompt）
  final String sampleLine;    // 代表性台词
  final Color? labelColor;    // 聊天室中角色标签颜色
}

class Message {
  final String id;
  final String role;          // 'user' | 'assistant'
  final String? characterId;  // 角色 id（assistant 消息需要）
  final String? characterName;// 角色显示名
  final String content;
  final DateTime timestamp;
  final bool isLoading;       // 是否正在加载中
}

class ChatRoom {
  final String id;
  final ChatMode mode;        // single | group
  final List<String> characterIds; // 参与的角色
  final List<Message> messages;
}

enum ChatMode { single, group }
```

## 9. API 配置管理

用户可在设置页配置：

- **API Key**（必填）
- **Model 名称**（默认：deepseek-v4-pro）
- 以上配置通过 SharedPreferences 本地持久化
